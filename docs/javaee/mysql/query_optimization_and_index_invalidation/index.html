    查询优化与索引失效 #

    查询优化 #

    应遵循一定规则，使得查询速度加快、数据库负担降低

    1. 避免查询不必要的数据，限制查询字段，不要取出全部列，尤其在多表联合查询时
    2. 联表查询时避免大表联小表
    3. 使用联合索引时遵循最左前缀原则
    4. 避免在代码中循环执行SQL
    5. 避免返回大量结果，要使用分页
    6. 考虑拆分大的复杂联表，改为多个简单查询在代码中复合
    7. 使用缓存，减少数据库重复访问
